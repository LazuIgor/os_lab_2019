int argc int argc - колличество аргументов, а char *argv[] - список аргументов


Препроцессор обрабатывает исходный код перед его компиляцией. Он выполняет текстовые замены, включает файлы, управляет макросами и условными директивами.
Компилятор преобразует обработанный препроцессором исходный код в объектный код.
Линковщик объединяет объектные файлы, созданные компилятором, в один исполняемый файл. Он также разрешает ссылки на функции и данные, которые находятся в разных модулях.

При статической линковке все необходимые библиотеки и их функции включаются непосредственно в исполняемый файл во время компиляции.
При динамической линковке исполняемый файл содержит только ссылки на библиотеки, которые загружаются в память во время выполнения программы.

-I Указывает директорию, в которой компилятор должен искать заголовочные файлы
-L Указывает директорию, в которой линкер должен искать библиотеки.
-l Указывает имя библиотеки, которую нужно линковать. Имя библиотеки должно быть указано без префикса lib и расширения .a или .so.
-shared Создает динамическую библиотеку.
-o Указывает имя выходного файла.
-c Компилирует исходный файл в объектный файл, но не выполняет линковку.
-fPIC Генерирует код, который может быть загружен в произвольное место в памяти (независимо от адреса). Это необходимо для создания динамических библиотек.

Команда ar позволяет использовать утилиту для архивации данных с соответствующим именем.


Переменная окружения LD_LIBRARY_PATH - это список имён каталогов, где следует искать библиотеки перед тем, как их будут искать по стандартным путям.
